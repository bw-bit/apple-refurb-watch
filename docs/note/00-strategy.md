# 売れるnoteの設計図(垢BAN安全版)

このドキュメントは、note本文を書く**前に**決めるべきことを全部決めるための設計図。
本文(`01-article-draft.md`)とX投稿(`02-x-posts.md`)は、すべてここから逆算して作られている。
公開前には必ず `03-compliance-checklist.md` を通すこと。

---

## 0. 前提:このnoteが「垢BANされにくい」理由

- **収益実績を一切売りにしない。** 「稼げる」系の訴求はゼロ。売るのは「欲しかったMacを定価より安く、人より早く買うための仕組み」という実用ノウハウ。
- **一次情報がすべて実在するコード。** このリポジトリ(`apple-refurb-watch`)が動いている事実だけで書けるので、盛る必要がない。
- 誇大表現・断定保証(「必ず買える」「100%通知が届く」)は本文・X投稿とも不使用。

## 2. 想定読者と「財布を開く理由」3行

### 想定読者

**「Apple整備済品を狙っているのに、いつ見ても在庫がない人」。**
エンジニアに限定しない。「コピペとGitHubの操作ができれば作れる」ことが価値。

サブ読者:プログラミング学習中で「実用的な題材でCLI/CI/CDを学びたい人」。

### 読者が悩みを感じる場面(具体)

- 朝、整備済ストアを見たら欲しいM系Macが出ていたのに、昼休みに見たら消えていた
- 在庫チェックのために1日に何度もストアを開く習慣がつき、疲れてやめた
- 市販の在庫通知サービスを探したが、Apple整備済品(日本)対応のものが見つからない/月額が高い
- 「スクレイピングで作れそう」と思ったが、サーバー代と常時稼働のさせ方で手が止まった

### 財布を開く理由(3行)

> 整備済品は「見た時に買う」ではなく「出た瞬間に知る」ゲームで、手動リロードでは構造的に勝てない。
> このnoteは、サーバー代0円・専用アプリなしで「出た瞬間にiPhoneに通知が来る仕組み」を、実際に動いているコードごと渡す。
> 有料部分はコピペで動く完全手順+自分の欲しいモデルだけに絞るカスタマイズ方法なので、読んだその日に自分専用の監視が動き出す。

## 3. 無料と有料の境界線

### 無料部分(信頼を作る)

1. 導入:整備済品が「買えない」のは運ではなく速度の問題、という問題の正体
2. 手動リロード・既存サービスがダメな理由(継続コスト・対応範囲)
3. 解決の方向性:「見に行く」のをやめて「来させる」
4. 仕組みの全体像(構成図レベル):GitHub Actions cron → スクレイピング → 差分検出 → Issue → GitHub Mobile通知
5. この構成を選んだ理由:サーバー0円/通知アプリ0円/Claude Codeでほぼ書いた、という技術選定の話

### 有料部分(具体的な解決策を渡す)

1. セットアップ完全手順(リポジトリ作成 → Actions権限 → iPhone通知設定 → 動作テストまで、コマンド付き)
2. コード全文と各ファイルの役割解説(`check.py` / `parser.py` / `watch.yml`)
3. 監視条件のカスタマイズ:`is_watched()` の書き換えレシピ集(例:MacBook Pro 64GB以上、iMacも監視、価格上限で絞る)
4. つまずきポイント集(通知が来ない/gh issue createが失敗/重複通知/Actionsが動かない)
5. 応用:Apple以外の「在庫が一瞬で消えるもの」への転用の考え方
6. Claude Codeに依頼したときの実際のプロンプトの流れ【要追記:あなたの体験】

### なぜここで切るのか

無料部分を読み終えた読者の頭の中は「仕組みは分かった。**で、自分の手元でどう動かすの?**」になっている。
「理解」は無料で全部渡し、「再現」を有料にする。
理解だけで自作できる上級者は元々買わない層なので、無料で全体像を見せても売上は毀損しない。
逆に「自分でも作れそうだが、ハマりどころを潰した完全手順に数百円払う方が早い」層が買う。

## 4. タイトル案10個

1. 欲しいMacが整備済品に出た瞬間、iPhoneに通知が来る仕組みを0円で作った
2. Apple整備済品を「見に行く」のをやめたら買えるようになった話
3. サーバー代0円。Apple整備済品の在庫監視ボットをGitHub Actionsだけで作る
4. 整備済品のMacが買えないのは運じゃなくて速度の問題だった
5. Claude Codeと作る、Apple整備済品の入荷通知システム(コード全文つき)
6. 1日に10回ストアを見る生活をやめた。在庫チェックはbotの仕事にした
7. M系Mac、定価で買う前に。整備済品を逃さない監視の仕組み(自作・0円)
8. GitHubのIssueをプッシュ通知に変える、いちばん安い在庫ウォッチャーの作り方
9. 「また売り切れてた」を最後にする。Apple整備済品ウォッチャー自作ガイド
10. 欲しい在庫は「出た瞬間に知る」——GitHub Actionsで作る自分専用アラート

**推し:1番。** 「欲しいMac」「出た瞬間」「iPhoneに通知」「0円」で、便益・即時性・コストが1行に入っている。数字や収益を使わずに強い。
