# 垢BAN回避チェックリスト

note公開前・X投稿前に、このリストを**毎回**通す。
1つでもチェックが付かない項目があれば、公開しない。

方針:このコンテンツは「収益ノウハウ」を売らない。売るのは実在するコードと再現手順。
盛る材料が最初から存在しない構成にしてあるので、以下は「うっかり」を防ぐための最終確認。

---

## A. note側(アカウント停止・記事非公開のリスク回避)

### A-1. 誇大・断定表現の排除

- [ ] 「必ず」「絶対」「100%」「誰でも」を、成果の保証として使っていない
- [ ] 「稼げる」「儲かる」系の訴求が本文・タイトル・目次に一切ない
- [ ] 「買える」と断定していない(「買い逃しにくくなる」「出た瞬間に知れる」に留める)
- [ ] 通知の速さは実測ベースの「最大5〜7分」のみ。「即時」「リアルタイム」と盛っていない

### A-2. 実績・数字の正確性

- [ ] 本文中のすべての数字に出典がある(コード・設定ファイル・実ログのいずれか)
- [ ] 他人の実績(参考記事の「累計2億円」等)を自分の文脈に流用していない
- [ ] 未確認の情報には「未確認」または「〜と言われる」と明記している
- [ ] 販売部数・売上を公表する場合は、スクリーンショット等で裏取りできるものだけ

### A-3. 有料部分の提供価値(返金請求・通報の予防)

- [ ] 無料部分で予告した内容(全手順・コード全文・カスタマイズ・つまずき集)が有料部分に**すべて**入っている
- [ ] 【要追記】プレースホルダが1つも残っていない(`grep 要追記` で確認)
- [ ] 「公開時にコード全文を貼る」のマーカーがすべて実コードに差し替わっている
- [ ] 有料部分の手順を、まっさらなアカウントで最初から実行して動作確認した

### A-4. コンテンツの性質

- [ ] スクレイピング対象(Apple)の利用規約への配慮を本文に明記している(低頻度アクセス・通知のみ・購入自動化はしない)
- [ ] 転売・買い占め用途を推奨する記述がない(むしろ明示的に禁止している)
- [ ] 外部への誘導(メルマガ・LINE・別商材)を有料部分の「中身」として使っていない
- [ ] note・Xの規約は変わるため、公開直前に最新の利用規約・ガイドラインを一読した(規約の条文を記憶で引用しない)

## B. X側(凍結・シャドウバンのリスク回避)

### B-1. 自動化・スパム

- [ ] 投稿はすべて手動(自動投稿ボット・一括予約連投を使っていない)
- [ ] 同一・ほぼ同一の文面を複数回投稿していない
- [ ] 他人のポストへのリプでnoteリンクをばら撒いていない
- [ ] 相互フォロー目的の機械的なフォロー/アンフォローをしていない

### B-2. リンクと頻度

- [ ] リンク付き投稿は10本中3本以内に収まっている
- [ ] 1日1本ペースを守っている(伸びないからと連投していない)
- [ ] 「RTで無料プレゼント」等のエンゲージメント釣り企画をしていない

### B-3. 内容の誠実さ

- [ ] 架空の質問・架空の購入者の声を作っていない(Day 8の注意書きを確認)
- [ ] 「買えた」実況は実際に起きた時のみ投稿する
- [ ] 投稿の主張がnote本文と矛盾していない

## C. 公開直前の最終動作

1. `grep -rn "要追記" docs/note/` → ヒット0件であること
2. `grep -rn "公開時にコード全文" docs/note/01-article-draft.md` → ヒット0件であること
3. 本文の技術記述をコードと最終突き合わせ(cron間隔・監視条件・通知経路)
4. note下書きプレビューをスマホで開き、無料/有料の境界位置を実機確認
5. 価格・返金設定(noteの返金申請の可否設定)を確認してから公開ボタンを押す
