# X勝ちパターン playbook

分析で確認できた再現可能な勝ちパターンだけを載せる資産ファイル。
`xa.py playbook` が毎回更新する。根拠の薄いものは末尾の「仮説」に隔離。

## 本則

### 法則1: 記事はタイトル・サムネ・冒頭3行で9割決まる
- 読者はタイムラインでタイトルとサムネを見てブクマするだけ。本文が読まれるのは勝負が決まった後
- 本文はAIに任せ、浮いた時間をタイトル・サムネ・冒頭3行に全投入する
- 根拠: 初期シード（元記事著者の実測）／最終確認日: 2026-07-17

### 法則2: ツリーは親ポストに結論を1本だけ置く
- 数字の塊・ノウハウの中身は2投稿目以降に降ろす
- 親は「落差 + 希少性」を1文に圧縮して続きに引っ張る（例: 800人で月500万 × 9割やらない）
- 根拠: 初期シード／最終確認日: 2026-07-17

### 法則3: 引用（セルフ引用含む）で記事を伸ばす・敗者復活させる
- Xは記事を「いい引用とセット」でタイムラインに流す。引用が記事の顔になる
- 記事を出したら必ず短い引用1〜2行。初速が悪ければ半日後にもう1本
- 引用は感想ではなく「元記事に飛びたくなる案内板」に徹する
- 根拠: 初期シード（元記事2万インプに対し引用1.8万インプ）／最終確認日: 2026-07-17

### 法則4: タイトルと一文目に「数字・固有名詞・断定」を詰める
- 数字は丸めず端数で（「1000人」より「1,037人」）
- 固有名詞は読者が今追っている言葉（Fable5 / ClaudeCode / note など）
- まず言い切る。補足は次の行で
- 根拠: 初期シード／最終確認日: 2026-07-17

### 法則5: 旬のワードに自分の話を乗せ、思いついた日に出す
- 今一番回っているワード（新機能・値上げ・延長・新ツール）× 自分の得意な話
- タイトルの頭に旬ワードと期限を置く（【7月19日まで】等）
- 旬は一瞬で腐る。完璧より速さ
- 根拠: 初期シード／最終確認日: 2026-07-17
